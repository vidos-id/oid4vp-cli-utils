## Highlights

- Refactor `@vidos-id/issuer` to use a storage-agnostic API where grants, access tokens, and nonces are explicit records passed through the issuance flow.
- Add filesystem-backed issuer state persistence in `@vidos-id/issuer-cli` so the CLI stores grants, access tokens, and nonces under the issuer directory.
- Update issuer examples, tests, and docs to reflect the new explicit-state workflow.

## New Features

- Export issuer record schemas and types for pre-authorized grants, access tokens, and nonces from `@vidos-id/issuer`.
- Return updated state records from exchange, proof validation, and issuance methods so callers can persist them in any storage layer.
- Add `FileSystemIssuerStorage` to the CLI package for filesystem-backed issuer runtime state.

## Other Changes

- Tighten the CLI issue flow around `--issuer-dir` in line with the new greenfield storage model.
- Refresh the end-to-end demo and package README examples to show explicit persistence of issuer state.
