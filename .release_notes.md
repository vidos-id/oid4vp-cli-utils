## Highlights

- Add Token Status List support across `@vidos-id/issuer`, `@vidos-id/wallet`, and `@vidos-id/wallet-cli`.
- Let issuers allocate credential status references, update status values, and publish signed `statuslist+jwt` documents without prescribing persistence.
- Let wallets resolve credential status on demand and expose it through `wallet-cli show --resolve-status`.

## New Features

- Add issuer-side status list APIs for creating lists, allocating indices, updating token status values, encoding compressed lists, and signing `statuslist+jwt` responses.
- Embed `status.status_list` references in issued credentials so relying wallets can resolve token state later.
- Add wallet support for fetching, verifying, decompressing, and interpreting OAuth Token Status Lists on demand.
- Add `wallet-cli show --resolve-status` to inspect live credential status from stored credentials.

## Bug Fixes

- Preserve issuer verification material alongside imported credentials so status list JWTs can be verified later.

## Other Changes

- Expand issuer, wallet, and wallet-cli documentation and tests for credential status support.
